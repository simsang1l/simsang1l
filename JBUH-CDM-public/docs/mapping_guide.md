# ETL 매핑 가이드 — 원본 코드 → OMOP-CDM 도메인 분류

병원 EMR 데이터를 OMOP-CDM으로 변환할 때, 원본 코드를 어떤 CDM 테이블(도메인)에 적재할지 결정하는 분류 방법론을 정리한 문서입니다.

> ⚠️ 원내코드 매핑 테이블은 병원 내부 정보를 포함하고 있어 전체를 공개하지 않습니다.
> EDI 분류 기준과 원내코드 분류 접근 방식을 중심으로 서술합니다.

---

## 분류 기준

```
1순위: EDI 분류번호 및 EDI 대분류코드 기준으로 도메인 결정
2순위: EDI가 없거나 분류번호가 없는 경우 → 원내코드 체계 기반으로 도메인 결정
```

---

## 대상 CDM 도메인

원본 데이터는 아래 4개 OMOP-CDM 도메인 중 하나로 분류됩니다.

| CDM 도메인 | 설명 | 예시 |
|-----------|------|------|
| **Measurement** | 검사 결과가 포함된 정량적 데이터 | 진단검사, 기능검사, 초음파 검사 |
| **Procedure** | 의료 행위 및 시술 | 처치, 수술, 마취, 영상촬영 |
| **Drug** | 약품 투약 관련 데이터 | 약제, 주사제, 수혈 관련 약품 |
| **Device** | 치료 재료 및 의료 기기 | 임플란트, 치료재료, 산소 |

---

## 1단계: EDI 코드 기반 분류

EDI(전자데이터교환) 분류번호가 존재하는 경우, 장(章)·절(節) 단위의 대분류코드를 기준으로 도메인을 결정합니다.

### 급여 항목

| 항목명 | 절 | EDI 대분류 | CDM 도메인 |
|--------|---|-----------|-----------|
| 기본진료료 | 가/(가 | A | Procedure |
| 검체검사료 | 나0 | B0001 | Measurement |
| 검체 검사료 | 누 | D/CZ211 | Measurement |
| 기능 검사료 | 나/너 | E/F | Measurement |
| 초음파 검사료 | 나9 | EB | Measurement |
| 내시경, 천자 및 생검료 | 나7 | E | Procedure |
| 영상진단 및 방사선치료료 | 다/더 | G/H | Procedure |
| 투약및 조제료 | 라 | J/30 | Procedure |
| 주사료 | 마 | X | Procedure |
| 마취료 | 바/버 | L | Procedure |
| 이학요법료 | 사/서 | M | Procedure |
| 정신요법료 | 아 | N | Procedure |
| 처치 및 수술료 | 자/저 | M/N/O/R/P/Q/T/S | Procedure |
| 치과 처치·수술료 | 차/처 | U | Procedure |
| 전혈 및 혈액성분제제료 | 파 | X | Device |

### 비급여 항목

| 항목명 | 절 | EDI 대분류 | CDM 도메인 |
|--------|---|-----------|-----------|
| 검체검사료 | 너 | CY | Measurement |
| 검체 검사료 | 노 | CZ/BZ | Measurement |
| 기능 검사료 | 노 | EZ/FZ | Measurement |
| 내시경, 천자 및 생검료 | 노 | EZ | Procedure |
| 초음파 검사료 | 노 | EZ | Measurement |
| 영상진단 및 방사선치료료 | 도 | GZ/HZ | Procedure |
| 주사료 | 모/머 | KZ/XZ | Procedure |
| 마취료 | 보 | LZ | Procedure |
| 처치 및 수술료 | 조 | SZ/PZ/QZ/RZ/OZ | Procedure |
| 치과 처치·수술료 | 초 | UZ | Procedure |

**설계 시 고려 사항**

- 급여 항목과 비급여 항목은 EDI 코드 체계가 다르므로 별도의 분류 테이블을 적용
- 동일 장(章) 내에서도 절(節)에 따라 도메인이 달라지는 경우 존재
- 일부 항목은 EDI 대분류코드가 여러 도메인에 걸쳐있어 세분화 규칙 필요

---

## 2단계: 원내코드 기반 분류

EDI 코드가 없는 경우, 병원 자체 코드 체계를 분석하여 도메인을 결정합니다.

**분류 접근 방식**

```
원내코드 분석
  ├── 코드 접두사(첫 글자) → 행위 유형 파악 → 기본 도메인 결정
  └── 코드 접미사 → 예외 케이스 확인 → 도메인 보정
```

**핵심 설계 원칙**

- 코드 접두사로 행위 유형(약제, 검사, 처치, 재료 등)을 1차 분류
- 접미사에 따라 동일 접두사라도 도메인이 달라지는 예외 처리
- 심사과를 통해 확인한 원내코드 분류 기준과 추가 확인된 코드를 반영

### 예시: 약제 코드 분류

약제 관련 원내코드는 하나의 접두사 내에서 **행위료**와 **약품 자체**가 혼재하는 대표적인 케이스입니다.

```
약제 코드 (D)
  ├── D1, D2, D3 (약제 행위료) → Procedure
  │     조제, 투약 등 의료 행위에 해당
  └── 나머지 D (약품 자체)    → Drug
        실제 투여된 약품 정보에 해당
```

이처럼 동일 접두사라도 세부 코드에 따라 도메인이 달라지므로, ETL 변환 시 접미사 패턴에 따른 조건 분기 로직을 적용했습니다.

> 전체 원내코드 매핑 테이블은 병원 내부 정보를 포함하고 있어 공개하지 않습니다.

---

## 예외 처리 사례

실제 매핑 과정에서 발생한 주요 예외 케이스와 해결 방법입니다.

### 1. 동일 코드 체계 내 도메인 분기

- **문제**: 하나의 코드 접두사가 여러 도메인에 걸쳐있음
- **해결**: 코드 접미사 패턴에 따른 조건 분기 로직 추가

### 2. EDI 코드 누락 데이터

- **문제**: 일부 레코드에 EDI 분류번호가 없거나 비표준 형식
- **해결**: 2단계 원내코드 기반 분류로 폴백(fallback), 그래도 분류 불가 시 수동 검토 대상으로 분리

### 3. 급여/비급여 분류 기준 차이

- **문제**: 동일 행위라도 급여·비급여에 따라 EDI 대분류코드가 다름
- **해결**: 급여/비급여 각각의 매핑 테이블을 별도 관리하고, 구분 필드를 기준으로 적용 테이블을 선택

---

## 검증

매핑 결과는 OHDSI DQD(Data Quality Dashboard)를 활용하여 검증하며, 주요 검증 항목은 다음과 같습니다.

- 도메인별 레코드 수 분포가 기대치에 부합하는지 확인
- 매핑되지 않은(unmapped) 레코드 비율 모니터링
- 원본 대비 CDM 적재 건수 일치 여부 확인
